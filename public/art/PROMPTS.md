# Art pipeline — VIBE CITY

All key art is generated by `scripts/generate-art.sh`, which drives the Codex
CLI's built-in image generation tool with a shared style block (painterly
open-world key art, pastel Miami palette, robot house style). Outputs land in
`/tmp/codex-art/*.png` and are post-processed into this directory as JPG.

The site loads each file via the `ArtImage` drop-in slot and falls back to the
in-repo vector/painted illustration when a file is missing, so partial art
sets never break the page.

| File | Slot |
| --- | --- |
| `hero.jpg` | Hero key art (logo space top-middle, dark bottom fifth) |
| `scene-construction.jpg` | Place panel: Under Construction (wireframe half-built city) |
| `scene-downtown.jpg` | Place panel: Downtown (neon night after rain) |
| `scene-canals.jpg` | Place panel: The Canals (boat chase, golden hour) |
| `scene-beach.jpg` | Place panel: North Beach (robots idling) |
| `dossier-foreman.jpg` | Crew dossier: The Foreman (character right third) |
| `dossier-architect.jpg` | Crew dossier: World Architect |
| `dossier-physics.jpg` | Crew dossier: Physics Lead |
| `dossier-missions.jpg` | Crew dossier: Mission Designer |
| `dossier-traffic.jpg` | Crew dossier: Traffic Director |
| `dossier-swarm.jpg` | Crew dossier: QA Swarm |
| `dossier-generic.jpg` | Crew dossier fallback for unknown live agents |

To regenerate any asset: delete it from `/tmp/codex-art/` and re-run
`./scripts/generate-art.sh` (existing files are skipped), then re-run the
post-processing step (see README "Regenerating the art").

Exact prompts live in `scripts/generate-art.sh`. Keep the STYLE and ROBOTS
blocks when adding new assets so the set stays coherent. Dossier images put
the character in the right third with a calmer, darker left side for the text
overlay.
